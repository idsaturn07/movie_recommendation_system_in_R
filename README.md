# 🎬 CineMatch — Movie Recommendation System in R

A content-based hybrid movie recommender built in R with a Shiny web interface. CineMatch analyses genre, plot keywords, and metadata to surface personalised film recommendations, and learns from your star ratings in real time.

## ✨ Features

- 🔍 Instant fuzzy search across 3,000+ movies
- 🎯 Content-based recommendations using TF-IDF + cosine similarity
- ⭐ Rate movies (1–5 stars) for personalised suggestions
- 🖼️ Live posters and backdrops via TMDB API
- 📊 Top Rated and Most Popular shelves on the home screen
- 🎬 Movie detail modal with overview, genre, score, and "More Like This"

## ⚙️ How It Works

### 1. Data Preprocessing
- Loads raw TMDB metadata
- Removes rows with missing titles, genres, or overviews
- Converts and validates numeric columns
- Deduplicates by title (keeps highest vote count)
- Engineers a composite `final_score` from popularity, rating, and vote count

### 2. Model Building
- Parses genre names from Python-style dict strings in the CSV
- Builds a content feature: `genres × 2 + keywords + tagline + overview`
- Applies TF-IDF (min 2 docs, word length 3+, noise words removed)
- Computes full pairwise cosine similarity matrix

### 3. Recommendations
For a query movie, candidates are scored as:
| Signal | Weight |
|---|---|
| Cosine similarity (TF-IDF content) | 55% |
| Jaccard genre similarity | 40% |
| Normalised final score (quality) | 5% |

### 4. Rating-Based Recommendations
Aggregates a weighted TF-IDF profile from all rated movies (weight = stars / 5), then ranks unseen movies by cosine similarity to that profile.

## 🚀 Setup

### 1. Install R packages
```r
install.packages(c("shiny", "dplyr", "httr", "jsonlite",
                   "tm", "proxy", "ggplot2", "scales", "ggrepel"))
```

### 2. Get the dataset

Download `movies_metadata.csv` from Kaggle:
👉 [TMDB Movies Dataset](https://www.kaggle.com/datasets/rounakbanik/the-movies-dataset)

Place it in a `data/` folder inside the project.

### 3. Generate the cleaned dataset

Open `notebooks/preprocessing.Rmd` in RStudio and run all chunks (`Ctrl+Alt+R`).
This generates `data/movies_final.csv`.

### 4. Run the app
```r
shiny::runApp("path/to/movie_recommender")
```

## 📌 Key Design Decisions

| Decision | Reason |
|---|---|
| TF-IDF bounds `c(2, Inf)` | With 3,000 movies, requiring 5+ appearances strips too many meaningful words |
| Popularity log-transformed | Raw popularity is heavily skewed — log scale compresses outliers |
| Genres weighted ×2 | Strongest signal for user preference; overview alone is too noisy |
| Final score clamped ±2 | Prevents viral blockbusters from dominating every recommendation |

---

## ⚠️ Limitations

- Content-based only — no collaborative filtering
- Cold-start problem for movies with very short overviews
- Posters not cached — TMDB fetches add latency on large shelves
- Model rebuilds on every app restart
